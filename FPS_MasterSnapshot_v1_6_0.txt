Face Prompt Studio（FPS）は Stable Diffusion / ComfyUI 向けの顔プロンプト
（face prompt）最適化・管理フレームワーク。

設計方針:
  - Clean Architecture — fps-core はアダプター依存ゼロ
  - ComfyUI はアダプターの一つ（差し替え可能）
  - 辞書・ルールはすべて JSON で定義（Python 修正不要）
  - テスト駆動開発（1122件 PASS）

現在バージョン: v1.6.0（2026-07-01）

  7-1. IssueType（7種）
    CONFLICT          同一排他グループ内の矛盾
    REDUNDANT         意味的重複
    MISSING_COVERAGE  重要カテゴリ未指定
    WEIGHT_IMBALANCE  重みの著しい偏り
    CROSS_CONFLICT    ★M6-1 positive/negative クロス矛盾
    STYLE_CONFLICT    ★v1.5 スタイル一貫性の問題
    TOKEN_BUDGET      ★v1.5 トークン数バジェット警告

  7-2. QualityScore（7フィールド）
    coverage_score         重要カテゴリ網羅度（0-100）
    balance_score          重みバランス（0-100）
    redundancy_score       非冗長性（0-100）
    overall_score          総合スコア（0-100）
    negative_coverage_score ★M6-1 ネガティブ網羅度（default=0.0）
    combination_score      ★v1.5 スタイル組み合わせ（default=100.0）
    token_score            ★v1.5 トークンバジェット余裕度（default=100.0）

  7-3. WEIGHTS（v1.5 5項目体制）
    coverage:    0.25
    balance:     0.20
    redundancy:  0.25
    combination: 0.20  ★v1.5
    token:       0.10  ★v1.5

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  8-1. CacheManager API
    get(namespace, key)        → value | None
    set(namespace, key, value, ttl=None)
    has(namespace, key)        → bool
    delete(namespace, key)     → bool
    clear(namespace=None)
    get_lookup(tag)            辞書ルックアップ用ショートカット
    set_lookup(tag, value)
    get_prompt(prompt)         プロンプトキャッシュ用ショートカット
    set_prompt(prompt, value)
    statistics()               → dict（hit_rate 等）
    evict_expired()            → int（削除件数）
    from_config(config)        → CacheManager（コンストラクタ）

  8-2. BackupManager API
    setup()                    バックアップディレクトリ初期化
    backup(target)             ファイルコピー方式で 1ファイル=1エントリ
                               entry.id = バックアップファイル名
                               （例: rules_20260701_075223_abc.json）
    restore(entry_id)          ファイルを元の場所へコピー復元
    list_backups(target=None)  新しい順
    get(entry_id)              → BackupEntry（なければ例外）
    get_or_none(entry_id)      → BackupEntry | None
    delete(entry_id)           → bool
    cleanup()                  max_count 超過分を削除 → int
    statistics()               → {total_backups/total_bytes/total_kb/by_target}

    BackupResult.errors        後方互換プロパティ（ComfyUI ノード対応）

  ■ DictionaryManager CRUD 注意事項
    - add_user_entry() はキーとエイリアスを正規化（スペース→アンダースコア）
    - delete_user_entry() は entries が空になると custom.json を削除

    - all_keys() は公開 API 未実装。

    _rules_cache はモジュールグローバル変数

    （test_v1_5_ai_scoring.py の @pytest.fixture(autouse=True) 参照）

  ■ QualityScore の後方互換性
    QualityScore は slots=True dataclass で positional 4引数で構築できる
    （5〜7番目のフィールドは全て default 値あり）
    既存テスト・ノードは追加フィールドなしで動作する

  ■ Web UI の showTab() 関数
    Knowledge ('kb') と History ('history') タブは
    main 要素を隠してフルWidth パネルを表示する方式
    新しいフルWidthタブを追加する場合は showTab() の条件分岐を拡張する

    entry.id = バックアップファイル名
